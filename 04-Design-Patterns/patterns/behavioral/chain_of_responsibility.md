# Chain of Responsibility Design Pattern

The Chain of Responsibility pattern is a behavioral design pattern that allows multiple objects (handlers) to process a request, but only one (or none) of them handles it. The request is passed along a chain of handlers until one of them processes it or it reaches the end of the chain.

## Key Points
- Decouples sender and receiver.
- Each handler decides either to process the request or to pass it to the next handler.
- Useful for event handling, logging, and approval workflows.
