# Design Patterns

Design patterns are proven solutions to common problems in software design. They provide reusable templates for solving recurring design challenges and help create more maintainable, flexible, and robust code.

### ğŸ—ï¸ Creational Patterns
Focus on object creation mechanisms, providing flexible ways to create objects without exposing creation logic.

**Patterns:**
- **Singleton** - Ensure only one instance exists
- **Factory Method** - Create objects without specifying exact classes
- **Abstract Factory** - Create families of related objects
- **Builder** - Construct complex objects step by step
- **Prototype** - Create objects by cloning existing instances

### ğŸ›ï¸ Structural Patterns
Deal with object composition and relationships, helping you build flexible and efficient object structures.

**Patterns:**
- **Adapter** - Make incompatible interfaces work together
- **Decorator** - Add new functionality to objects dynamically
- **Facade** - Provide a simplified interface to complex subsystems
- **Composite** - Treat individual and composite objects uniformly
- **Proxy** - Provide a placeholder for another object
- **Bridge** - Separate abstraction from implementation

### ğŸ­ Behavioral Patterns
Focus on communication between objects, providing effective ways for objects to interact.

**Patterns:**
- **Observer** - Define one-to-many dependency between objects
- **Strategy** - Encapsulate algorithms and make them interchangeable
- **State** - Allow objects to alter behavior when internal state changes
- **Chain of Responsibility** - Pass requests along a chain of handlers
- **Mediator** - Define how objects interact with each other
- **Command** - Encapsulate requests as objects
- **Template Method** - Define algorithm skeleton with customizable steps
- **Iterator** - Provide sequential access to elements
- **Visitor** - Separate algorithms from object structure
- **Memento** - Capture and restore object state
