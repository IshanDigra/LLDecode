# LLD Problems

A comprehensive collection of Low-Level Design problems organized by difficulty level. Each problem includes problem statement, solution approach, implementation, and learning outcomes.

### 01-Beginner
**Focus:** Basic concepts, simple systems
- **Entities:** 2-3 core entities
- **Complexity:** Basic CRUD operations
- **Patterns:** 0-1 design patterns
- **Examples:** Traffic Control, Task Management

### 02-Easy  
**Focus:** Simple systems with clear requirements
- **Entities:** 3-5 core entities
- **Complexity:** Basic state management
- **Patterns:** 1-2 design patterns
- **Examples:** Parking Lot, Vending Machine, Logging Framework

### 03-Medium
**Focus:** Multiple interacting services
- **Entities:** 5-8 core entities
- **Complexity:** State management, some concurrency
- **Patterns:** 2-3 design patterns
- **Examples:** LRU Cache, Elevator, ATM, Hotel Management

### 04-Hard
**Focus:** Complex business logic and multiple services
- **Entities:** 8+ core entities
- **Complexity:** Complex business rules, payment systems
- **Patterns:** 3+ design patterns
- **Examples:** BookMyShow, Splitwise, Spotify, Food Delivery

### 05-Expert
**Focus:** Enterprise-level systems with scalability
- **Entities:** 10+ entities with complex relationships
- **Complexity:** Advanced algorithms, real-world constraints
- **Patterns:** Multiple advanced patterns
- **Examples:** Amazon, Uber, Chess, Course Registration
